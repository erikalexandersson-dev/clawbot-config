# Memory Structure

Purpose: keep notes organized by domain so retrieval is faster and cleaner.

## Folders
- `memory/inbox/` — quick captures before sorting
- `memory/calendar/` — calendar-related notes, bookings, date checks
- `memory/health/` — medical/health tracking and plans
- `memory/flying/` — paragliding/flying notes and research
- `memory/integrations/` — API keys setup notes, channels, auth workflows (no raw secrets)
- `memory/automation/` — cron/heartbeat/tooling workflows
- `memory/finance/` — banking/admin reminders and references
